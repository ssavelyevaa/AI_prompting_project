# AI_prompting_project: Prompt Structure & Usability

## Overview  
This mini-project explores how prompt structure and context influence the clarity, relevance, and usability of AI-generated outputs in common knowledge work tasks.

Rather than focusing on model performance, the goal was to understand how **input design affects output usability** — specifically, how easily a person can understand and act on the result.

---

## Goal  
To evaluate how different levels of prompt structure impact:

- clarity  
- organization  
- usefulness  
- actionability  
- alignment with task intent  

---

## Tasks Tested  

Three common knowledge work tasks were used:

1. **Summarizing a complex text**  
2. **Transforming messy notes into a structured guide**  
3. **Generating a study workflow for a specific topic**  

---

## Method  

For each task, outputs were generated using the same AI model and compared across three prompt types:

1. **Unstructured prompts**  
   Minimal instruction  
   > e.g., "Summarize this text"

2. **Lightly guided prompts**  
   Basic direction without strict constraints  
   > e.g., "Summarize this in simple terms"

3. **Structured prompts**  
   Clearly defined format and expectations  
   > e.g., specifying sections, steps, or output structure  

For the workflow task, **task-specific context** (neuroscience topics) was included to evaluate how context influences output relevance.

Outputs were evaluated qualitatively across:
- clarity  
- structure  
- usefulness  
- actionability  
- alignment with task intent  

---

## Key Observations  

### 1. Outputs are generally accurate, but not always usable  
Unstructured prompts often produced correct information, but responses were:
- longer than necessary  
- less focused  
- harder to scan quickly  

---

### 2. The model defaults to a "generic helpful answer"  
Without clear structure, the model tends to:
- include extra explanations  
- add general advice  
- interpret the task broadly rather than precisely  

---

### 3. Structure improves precision and usability  
Structured prompts consistently produced outputs that were:
- more concise  
- better organized  
- directly aligned with the task  
- easier to use immediately  

---

### 4. Context improves relevance, but structure is still necessary  
Providing task-specific context (e.g., neuroscience topics) improved relevance, but without structure:
- outputs still included unnecessary or generic information  

The most effective results came from combining:
- **context (for relevance)**  
- **structure (for usability)**  

---

## Key Insights  

> The biggest difference wasn’t whether the model could produce a correct answer — it was whether the output was immediately usable.

- **Structure improves usability and efficiency, not just accuracy**  
- **Context improves alignment with the task**  
- **Without structure, the model "guesses" the intended format**  
- **Prompting is less about asking questions and more about designing outputs**  

---

## Why This Matters  

In real-world workflows, AI outputs need to be:
- quick to understand  
- easy to act on  
- aligned with specific tasks  

Even high-quality outputs can create friction if they:
- require extra interpretation  
- include unnecessary information  
- lack clear structure  

This project reflects an interest in **AI enablement** — not just what AI can generate, but how to make its outputs usable, structured, and effective in real work environments.

---

## Reflection  

This project shifted my perspective on AI from a tool that generates answers to a system that requires **intentional input design**.

It highlighted that:
- usability depends heavily on how tasks are framed  
- small changes in prompt structure can significantly improve outcomes  
- designing workflows and constraints is often more important than the model itself
